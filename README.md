## 反映したMTG論点

### 5/20 MTG

- なぜ効いたか分からない → 要素分解＋多軸評価
- セグメント×クリエイティブが見えない → 4軸の評価軸
- 訴求の使い回しによる画一化 → 認知ポートフォリオの偏り検出
- 認知ファネルの役割整理がない → ファネル段階別KPI
- エンゲージメント単価評価で価格訴求が勝つ → ファネル別評価指標の提案
- 売りたい商品軸でキャンペーン → 「狙う認知」起点の企画フロー

### 6/11 対面会（UGC相談）

- UGC施策の効果を測りたいが投稿の目視確認が追いつかない → LLMによる投稿内容の自動集約・テーマ分類・件数把握
- どういった内容の投稿が多いか知りたい → テーマ別件数ランキング + ネガポジ分析
- UGCは独立ツールにしない → 広告と同じ4軸タグで統合し、UGC発の訴求をクリエイティブ生成に還流（統合管理画面）
- X API / Instagram Graph API の契約・権限・収集範囲・頻度は要先方確認（`dashboard_ugc.html` にアラート明記）

## 注意事項

- データは全て仮（業界感のレンジ）
- 案件名・コピー・サムネ・UGC投稿文は実在のものではない（実在の投稿・ユーザーとは無関係）
- X / Instagram の API仕様・費用は確定情報として記載しない（要先方確認アラートに集約）
- 5/24デモまでに、先方提供データで第1層を実体化する想定（本モックは握り用）
